Script Performance Fundamentals

Cache Components
GetComponent() is expensive. Never call it inside Update(). Store references in Awake() or Start().
Use CompareTag()
tag == "Player" creates garbage memory allocation. Use CompareTag("Player") which is zero-allocation and faster.
Avoid String Concatenation
"Score: " + score creates a new string every frame. Use StringBuilder or update UI only when values change.
90x
Update() runs 90 times per second.
Small mistakes multiply fast.
The "Update" Trap
void Update() { // BAD: Searching every frame Rigidbody rb = GetComponent<Rigidbody>(); rb.AddForce(Vector3.up); // BAD: Allocating memory if (gameObject.tag == "Enemy") { ... } }
The Optimized Way
Rigidbody _rb; void Awake() { // GOOD: Cache once _rb = GetComponent<Rigidbody>(); } void Update() { _rb.AddForce(Vector3.up); // GOOD: Zero allocation if (gameObject.CompareTag("Enemy")) { ... } }
