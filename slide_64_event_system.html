Event System Optimization

The Polling Problem
Checking a condition every frame (Polling) is wasteful. If you have 100 enemies checking if (player.isDead) every frame, that's 9,000 useless checks per second (at 90 FPS).
The Event Solution
Use C# Events or Actions. The enemies don't check anything. When the player dies, the player script "shouts" once, and all enemies react immediately.
UnityEvents vs C# Events
UnityEvents (used in UI buttons) are slower and create garbage. For high-frequency code, use standard C# Action or delegate.
Bad: Polling in Update
void Update() { // Running 90 times per second... for nothing if (player.health <= 0) { Celebrate(); } }
Good: Event Driven
void Start() { // Subscribe once player.OnDeath += Celebrate; } void Celebrate() { // Only runs when it actually happens PlayAnimation("Dance"); }
MEMORY LEAK WARNING: If you subscribe with +=, you MUST unsubscribe with -= in OnDestroy(). If you forget, the object will never be garbage collected, causing a massive memory leak over time.
