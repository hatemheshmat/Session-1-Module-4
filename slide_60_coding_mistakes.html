Common Coding Mistakes

LINQ in Update
Memory Garbage
var active = list.Where(x => x.isOn).ToList();
LINQ creates hidden memory allocations every time it runs. Doing this in Update() triggers the Garbage Collector frequently, causing frame stutters. Use for loops instead.
Debug.Log Spam
Heavy I/O
Debug.Log("Player Pos: " + transform.position);
Logging involves string concatenation, stack trace capture, and writing to device logs. It is extremely slow. Remove all logs in release builds!
GameObject.Find
CPU Spike
var boss = GameObject.Find("Boss");
This iterates through every object in the scene comparing strings. It is a performance disaster if called every frame. Cache references in Start().
Empty Update()
Overhead
void Update() { /* TODO */ }
Even an empty Update() method has a cost (Native C++ to Managed C# context switch). If you don't need it, delete the method entirely.
Golden Rule: If it happens every frame (Update/FixedUpdate), it must be zero-allocation and O(1) complexity.
