Profiling Best Practices

Profile on Target Hardware

Never trust Editor FPS. Your PC is 10x-50x faster than a mobile chipset. Always build to the Quest to get accurate data.

Avoid "Deep Profile"

Deep Profile instruments every C# method, adding massive overhead. Only use it for specific script debugging, never for general performance checks.

Use Development Builds

Check "Development Build" and "Autoconnect Profiler" in Build Settings. This allows the Unity Profiler to attach to the running app over Wi-Fi or USB.

Monitor Thermal State

If the device gets too hot, it will throttle (slow down) the CPU/GPU. Ensure the device is cool before starting a performance capture.

The Golden Rule

If it runs fast in the Editor, it means nothing. If it runs slow in the Editor, it will be unplayable on Quest.
