Submission Requirements

To publish on the Meta Quest Store or App Lab, your application must pass a strict set of Virtual Reality Checks (VRCs). Failure results in automatic rejection.
Performance
VRC.Performance.1
Stable Frame Rate
Must maintain 72/90 FPS for the majority of gameplay. Frequent drops below target are grounds for rejection.
VRC.Performance.2
Load Times
App must display a splash screen or loading indicator if loading takes longer than 3 seconds. No black screens > 10s.
Functionality
VRC.Functional.1
Entitlement Check
App must verify the user owns the app via the Platform SDK within 10 seconds of launch.
VRC.Functional.2
System Events
App must handle the "Oculus Button" press (pause/menu) and headset removal (auto-pause) correctly.
Comfort
VRC.Comfort.1
No Forced Motion
The camera must never move or rotate without user input (except for teleportation). No "head locking" during cutscenes.
VRC.Comfort.2
Rating Accuracy
The app's comfort rating (Comfortable, Moderate, Intense) must match the actual experience.
