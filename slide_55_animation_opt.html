Animation System Optimization

The String Problem
Using strings to control animations (e.g., Play("Run")) forces Unity to hash that string every single time it's called. This creates CPU overhead and garbage collection spikes.
The Solution: Hashes
Calculate the hash ID once at startup (or compile time) and use the integer ID for all checks and updates. Integers are blazing fast compared to strings.
Tip: For simple movements (doors, spinning coins), don't use an Animator at all. Use a Tweening Library (like DOTween) or simple script rotation.
PlayerAnimation.cs
private int runHash;

void Start() {
  // Cache the hash ID once
  runHash = Animator.StringToHash("Run");
}

void Update() {
  // Use the integer ID (Fast!)
  if (isMoving)
    animator.SetBool(runHash, true);
}
