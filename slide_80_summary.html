Summary & Key Takeaways

Comfort First
Performance bugs in VR cause physical pain (nausea), not just annoyance.
Avoid sensory conflicts: Don't move the camera without user input.
Use comfort features like Tunneling and Snap Turn.
The Budget
Target 72 FPS (13.8ms) or 90 FPS (11.1ms) strictly.
Keep Draw Calls under 100-150 via Batching and Instancing.
Limit Polygon count (100k-300k visible) and use LODs.
The Workflow
Profile Early: Don't wait until the end of the project.
Profile on Device: Editor performance is a lie.
Bake Everything: Use lightmaps and occlusion culling to save runtime cost.
A smooth Low-Poly game beats a laggy Realistic one every time.
