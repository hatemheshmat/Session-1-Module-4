Performance Testing Workflow

1
Editor Check
Quickly validate logic and catch obvious errors (e.g., infinite loops, null refs).
Note: FPS here is meaningless.
2
Dev Build
Build Settings:
• Development Build: ON
• Autoconnect Profiler: ON
• Script Debugging: OFF (usually)
3
Runtime Check
Wear the headset.
Open OVR Metrics Tool overlay.
Play through the level naturally.
Watch for FPS drops below 72/90.
4
Deep Dive
If FPS drops:
• CPU Bound? Connect Unity Profiler.
• GPU Bound? Use RenderDoc or Snapdragon Profiler.
The Golden Rule: Never optimize based on Editor performance. The PC CPU/GPU is 10x-50x faster than the mobile chipset in the Quest. Always profile on the device.
