Dynamic Batching

For Moving Objects
Unlike Static Batching, this works for objects that move, rotate, or scale. Unity transforms their vertices on the CPU into world space, then draws them together.
The CPU Cost
Transforming vertices on the CPU is slow. If the mesh has too many vertices, the CPU time spent transforming them is higher than the time saved by batching the draw call.
Best Use Cases
  • Particle Systems
  • Simple UI Elements
  • Low-poly projectiles (bullets, debris)
