Async & Coroutines

VS
Coroutines
IEnumerator SpawnRoutine() { while (true) { SpawnEnemy(); // Allocates memory! yield return new WaitForSeconds(1f); } }
Best For
Visual sequences (Wait, then Move)
Frame-based timing
Downsides
Runs on Main Thread (Can stutter)
Garbage Allocation (new WaitForSeconds)
Async / Await
async Task LoadDataAsync() { // Runs on background thread await Task.Run(() => { HeavyCalculation(); }); // Back to main thread }
Best For
Heavy Logic (Pathfinding, I/O)
Clean return values
Downsides
Exception handling is tricky
Harder to sync with Unity frames
Optimization Tip: If you use Coroutines, CACHE your wait objects. Don't create new WaitForSeconds(0.1f) inside a loop. Create it once in Start() and reuse it to avoid Garbage Collection spikes.
